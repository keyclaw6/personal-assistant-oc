# Reframing Material Protocol

Use this when the user wants to see a belief from different angles.

## Purpose

Generate multiple frames so the user can discover which understanding actually changes preference.

## Required Input

- Belief statement.
- Recent example.
- Desired direction.
- Current cost.
- Protective function.

## Frames

### Preference Frame

What does this belief make seem like the best option?

### Protection Frame

What pain does this belief prevent?

### Cost Frame

What does this belief quietly cost?

### Evidence Frame

What evidence supports it, weakens it, or narrows it?

### Identity Frame

What kind of person does this belief imply the user is?

### Social Frame

How does this belief shape social action, status, honesty, intimacy, or conflict?

### Future Frame

If this belief remains unchanged for five years, what compounds?

### Daily-Life Frame

What would change tomorrow if the updated belief were genuinely understood?

### Experiment Frame

What real-world test would provide evidence?

### Book Lens

Which source or book gives a useful model for this belief?

## Output

Write reframing material into the session analysis or a belief file section called:

```text
## Reframing Material
```

Do not present every frame every time. Choose the frames most likely to create understanding.
