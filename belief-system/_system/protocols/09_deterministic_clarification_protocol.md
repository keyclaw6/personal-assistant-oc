# Deterministic Clarification Protocol

The deterministic clarification is the stable record later agents should trust most.

## Purpose

It turns a messy session into structured facts without adding coaching style, persuasion, or broad pattern claims.

## Independence

Whenever possible, a clarification should be generated by an agent that did not run the coaching session.

If the same agent must produce it, it should switch roles and use only:

- Transcript.
- Context-loaded file.
- Belief file before and after update.
- Explicit decisions and actions.

## Output File

```text
04_sessions/<session-id>/04_deterministic_clarification.json
```

Use:

```text
_system/schemas/session_clarification.schema.json
```

## Allowed Content

- Beliefs discussed.
- User claims.
- Concrete examples.
- Agent inferences with basis and confidence.
- Decisions made.
- Experiments created.
- Completion marks.
- Next actions.

## Disallowed Content

- Motivational coaching language.
- Hidden chain-of-thought.
- Unsupported diagnosis.
- Long-term pattern claims without cross-session evidence.
- Claims that the user has changed if only insight occurred.

## Confidence

- `low`: unclear or weakly supported.
- `medium`: reasonably supported by the session.
- `high`: explicit, repeated, or behaviorally anchored.
