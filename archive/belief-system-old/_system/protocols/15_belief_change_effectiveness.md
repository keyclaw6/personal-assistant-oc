# Belief Change Effectiveness Protocol

This protocol keeps the system focused on actual belief change rather than pleasant reflection.

## Belief Change Model

The system assumes a belief is changing when one or more of these shifts occur:

- Prediction changes.
- Preference changes.
- Cost and benefit perception changes.
- Identity implication changes.
- Social expectation changes.
- Action changes with less force.
- The old belief becomes narrower, less credible, or less emotionally necessary.

## Effectiveness Questions

For every active belief, ask:

1. What does the old belief make seem best?
2. What pain does it reduce?
3. What joy or benefit does it promise?
4. What new understanding would make a better option honestly preferable?
5. What daily-life behavior would change if the update were real?
6. What evidence would the user actually trust?
7. What experiment can generate that evidence?

## Anti-Surface-Reading Standard

For books and sources:

```text
If I actually understood this, then in normal life I would...
```

If this sentence cannot be completed concretely, the source has not yet been integrated.

## Integration Standard

Insight is not integration. Integration requires at least one:

- Changed prediction.
- Changed choice.
- Changed emotional tradeoff.
- Changed behavior.
- Clear contextual narrowing of the old belief.
- Explicit rejection of a belief candidate.

## Common Failure Modes

- The system summarizes instead of extracting lived implications.
- The agent creates too many candidate beliefs without prioritizing them.
- The user understands intellectually but avoids experiments.
- The agent overstates a pattern from too little evidence.
- The agent marks completion without user ownership.
- The system updates memory too broadly from one session.

## Correction Moves

- Convert summaries into belief candidates.
- Convert belief candidates into daily-life implications.
- Convert daily-life implications into experiments.
- Convert experiments into evidence updates.
- Convert repeated evidence into pattern reports.
- Convert stable integration into completion reviews.
