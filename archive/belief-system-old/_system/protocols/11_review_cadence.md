# Review Cadence

## After Every Session

- Update session folder.
- Update relevant belief file.
- Update experiments.
- Update progress ledger.
- Add new candidates if discovered.

## Weekly Review

Purpose:

- Check movement.
- Pick next belief.
- Review active experiments.
- Notice early patterns.

Use:

```text
_system/templates/weekly_review.md
```

## Monthly Review

Purpose:

- Run pattern analysis across deterministic clarifications.
- Rank next beliefs.
- Identify books or sources to revisit.
- Check whether any beliefs are ready for completion review.

## Quarterly Review

Purpose:

- Review belief architecture.
- See which beliefs moved from candidate to integrated.
- Identify root patterns.
- Export a progress snapshot.

## Completion Reviews

Run when:

- The user asks to mark a belief complete.
- A belief has stayed integrated across multiple contexts.
- A belief no longer creates useful work.

Use:

```text
_system/protocols/05_completion_protocol.md
```
