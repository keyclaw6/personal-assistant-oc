# PATTERN_WORK.md

Pattern work is an evidence discipline, not a vibe.

## Evidence levels

Strong evidence:

- explicit Kristian statement
- repeated journal entries
- repeated commitment behavior
- confirmed correction
- concrete calendar/task/email pattern

Medium evidence:

- similar theme across 2+ sessions
- repeated emotional wording
- recurring avoidance of the same decision

Weak evidence:

- one phrase
- model intuition
- resemblance to a framework
- a book idea that seems relevant

Weak evidence becomes a question, not a claim.

## Creation threshold

Create `memory/patterns/<slug>.md` only when one is true:

- Kristian confirms it.
- 3 observations across 2+ weeks.
- 2 observations plus clear practical cost.
- it directly affects an active commitment or decision.

## Pattern file shape

```md
# Pattern: <plain name>

Status: candidate | active | retired
Confidence: low | medium | high
First observed:
Last reviewed:
Tags:

## Hypothesis
One sentence.

## Evidence
- YYYY-MM-DD — source — concrete observation

## Counterevidence / cautions
- What could make this wrong?
- What Kristian rejected or corrected.

## Practical impact
What this affects in daily life.

## Useful experiment/question
One next step.

## Review log
- YYYY-MM-DD — update
```

No evidence list, no pattern.
