# BELIEF_WORK.md

Belief work is useful only when it changes real life.

## Use when

- Kristian names a belief.
- A belief blocks a practical decision.
- A repeated pattern suggests an experiment could update expectations.

## Questions

- What does Kristian currently expect to be true?
- Why does that expectation make sense?
- What does it protect?
- What does it cost?
- What alternative model would be more accurate or useful?
- What small real-world experiment could test the update?

## Belief file shape

```md
# Belief: <slug>

Status: noticed | working | testing | settled | archived
Started:
Last touched:

## Current belief
## Why it makes sense
## Cost
## Alternative model
## Experiment
## Evidence log
## Kristian corrections
```

Do not create belief files from clever interpretations. Use confirmation,
repetition, or practical cost as the threshold.
